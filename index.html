Presales Tracker
Connect a Supabase project to enable persistent storage and real-time collaboration across your team.
1
Create a free Supabase project
Go to supabase.com, sign up (free tier is plenty), and create a new project. Takes about 2 minutes.
2
Run this SQL to create the tables
In your Supabase project, go to SQL Editor and run:

create table deals ( id uuid primary key default gen_random_uuid(), data jsonb not null, updated_at timestamptz default now() ); create table settings ( key text primary key, value jsonb not null, updated_at timestamptz default now() ); alter table deals enable row level security; alter table settings enable row level security; create policy "public read" on deals for select using (true); create policy "public write" on deals for all using (true); create policy "public read" on settings for select using (true); create policy "public write" on settings for all using (true);
3
Find your API key
In your Supabase project: click Project Settings (gear icon, bottom-left) → click API in the left sidebar → under "Project API keys", copy the anon / public key. It starts with eyJ… and is about 200 characters long. Do not use the service_role key.

How the automation will work
→Every Friday at 5pm, HubSpot is queried for all deals owned by the presales team.
→Changed deal stages, new ARR values, and new comments are written back to this tracker automatically.
→A Slack message summarises what changed, flags KTLO/at-risk accounts, and lists deals requiring action.
→An optional email digest mirrors the Slack update in HTML format for stakeholders outside Slack.
→Manual overrides in this tracker are preserved and not overwritten by the sync.
